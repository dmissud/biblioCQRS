🔍 BiblioCQRS - API de Requête (Query)

Le Projet BiblioCQRS

BiblioCQRS est un système de gestion de bibliothèque moderne implémenté en utilisant l'architecture CQRS (Command Query Responsibility Segregation) et Event Sourcing.

L'architecture sépare strictement les opérations en deux parties distinctes : l'écriture (Command) et la lecture (Query), qui communiquent de manière asynchrone pour garantir performance et scalabilité.

Ce composant : Query

Ce module (biblio-query-boot) est responsable de la partie lecture.

Il écoute les événements produits par la partie commande (via un bus d'événements) et met à jour des vues optimisées pour la consultation (read models). Il expose ensuite des endpoints rapides permettant de consulter l'état du système en temps quasi-réel (ex: liste des livres disponibles, historique des emprunts).
