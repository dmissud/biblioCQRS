# Bienvenue sur la documentation de BiblioCQRS 📚

**BiblioCQRS** est un projet de gestion de bibliothèque basé sur une architecture **CQRS (Command Query Responsibility Segregation)** et **Event Sourcing**.

## Pourquoi cette documentation ?

Dans un projet aux principes architecturaux forts (Event-Driven, CQRS, DDD), il est essentiel d'avoir un référentiel commun pour :
1. Comprendre le domaine métier de la bibliothèque.
2. S'aligner sur les choix d'architecture (ADRs).
3. Fluidifier l'onboarding des nouveaux développeurs.

Cette documentation a été conçue en mode **"Docs as Code"**. Elle vit avec le code et est versionnée dans Git.

## Comment naviguer ?

Utilisez le menu situé dans la barre supérieure ou latérale pour explorer les différentes sections :

* **🎯 Exigences** : Le recueil des besoins, le glossaire métier et le backlog.
* **🏗 Architecture** : Les diagrammes C4, la modélisation des agrégats et l'organisation des topics Kafka.
* **⚖️ Décisions (ADRs)** : Les *Architecture Decision Records* justifiant nos choix techniques majeurs (CQRS, Kafka, etc.).
* **💻 Développement** : Le guide pour installer le projet en local, lancer le `docker-compose` et contribuer au code.
